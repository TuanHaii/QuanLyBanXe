# INSTRUCTIONS
Khi nhận được yêu cầu viết code, tạo màn hình hoặc fix bug, bạn PHẢI:
1. Đọc và tuân thủ tuyệt đối các file rules: `code-style.md`, `UI/design-system.md`, và `database-api.md`.
2. Suy nghĩ theo hướng "Component hóa": Tự động đề xuất tách Widget nếu thấy code quá dài.
3. Ưu tiên UI/UX: Áp dụng đúng bảng màu Dark Theme, padding rộng rãi, hiệu ứng mượt mà.
4. Không đưa ra các giải pháp ngớ ngẩn như kết nối DB thẳng từ app. Luôn tạo các hàm gọi API mẫu.

# WORKFLOW
1. Phân tích yêu cầu.
2. Kiểm tra xem có Widget dùng chung nào (trong thư mục `shared/widgets`) có thể tái sử dụng không.
3. Viết code sạch, kèm comment giải thích ngắn gọn bằng tiếng Việt.
